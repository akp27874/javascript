You're diving into one of the most important concepts in DOM event handling: **event propagation**, specifically the difference between **bubbling** and **capturing**.
üîÅ Event Propagation: Two Phases
When an event (like a click) happens on an element, it travels through the DOM in two phases:
1. **Capturing Phase** (aka "trickling down"): - Starts from the `window` and moves **down** to the target element. - You can listen to this phase by passing `true` as the third argument in `addEventListener`.
2. **Bubbling Phase**: - Starts from the target element and bubbles **up** to its ancestors. - This is the default phase when you don‚Äôt pass `true`.

#### üß± Parent & Child ```js parent.addEventListener('click', ...) // bubbling (default) child.addEventListener('click', ...) // bubbling (default) ``` - When you click the **Child** button: - First, `Child clicked` logs (target element). - Then, `Parent clicked` logs (bubbles up).

#### üß± Outer & Inner ```js outer.addEventListener('click', ..., true) // capturing inner.addEventListener('click', ...) // bubbling ``` - When you click the **Inner** button: - First, `Outer clicked during capture` logs (capturing phase). - Then, `Inner clicked` logs (target). - No bubbling to `outer` again, because it already handled the event in capture phase.

### üß† Why Use `true` (Capture)? Passing `true` lets you intercept the event **before it reaches the target**. This is useful when: - You want to **pre-process** or **block** events early. - You need to handle events **before child elements** do. - You‚Äôre building custom UI components or debugging event flow.
